# アラートに関する考慮事項

サービス・レベル・インディケータ(SLI)およびエラー・バジェットからアラートを生成するには、これらの2つの要素を特定のルールに統合する方法が必要です。
目標は、重大なイベント(エラー・バジェットの大部分を消費するイベント)について通知を受けることです。

アラート戦略を評価するときは、次の属性を考慮してください。

_精度_  
検出されたイベントのうち、重要である割合。
各アラートが重大なイベントに対応している場合、精度は100%です。
アラートは、トラフィックが少ない期間の重要でないイベントに特に敏感になる可能性があることに注意してください([86ページの「低トラフィックなサービスとエラー・バジェットのアラート」](../05_03_low-traffic-services-and-error-budget-alerting/README.md)で説明します)。

_リコール_  
検出された重要なイベントの割合。
重大なイベントが発生するたびにアラートが生成される場合、リコールは100%です。

_検知時間_  
さまざまな状況で通知を送信するのにかかる時間。
検知時間が長いと、エラー・バジェットに悪影響を与える可能性があります。

_リセット時間_  
問題が解決してからアラートが起動されるまでの時間。
リセット時間が長いと、混乱したり、問題が無視されたりする可能性があります。
